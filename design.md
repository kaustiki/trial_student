# Design Instructions

## Product Direction

The Student Care Referral System is an operational school tool. It should feel
calm, clear, and work-focused. Avoid marketing-style hero sections, decorative
cards, oversized type, and one-note color palettes.

## Interface Rules

- Use the app experience as the first screen after login.
- Keep page sections unframed or simple full-width areas; use cards only for
  repeated items, forms, and compact summaries.
- Use lucide icons for familiar actions such as save, submit, logout, dashboard,
  and referral navigation.
- Keep cards at `8px` radius or less.
- Use dense but readable layouts for dashboards and forms.
- Use stable dimensions for navigation, buttons, counters, and form controls so
  hover states and labels do not shift layout.
- Do not use visible helper text to explain obvious UI mechanics.
- Do not use gradient orbs, bokeh blobs, or purely decorative background art.

## Color Direction

Use a restrained palette with neutral surfaces and a few semantic accents:

- Ink: primary text
- Ocean: primary action and navigation
- Leaf: success/submit actions
- Clay: warning/escalation accents
- Paper: app background

## Accessibility

- Buttons with icons need accessible labels or visible text.
- Inputs need labels.
- Text must not overlap or overflow its container on mobile or desktop.
- Keep focus states visible.
